7 · Speaking the Machine's Language

We have our intermediate representation. Clean, simple, low-level. Instructions like %0 = add %1, %2 that operate on virtual registers and jump between basic blocks. It's almost assembly already — except it isn't. No real processor understands virtual registers. No real processor executes br_cond %0, b1, b2.

We need to translate IR into something a real CPU can execute. For us, that means x86-64 assembly. This is the moment where all our careful abstractions meet the hard reality of a physical machine.

This chapter is different Everything before this was about transforming the source program while keeping it abstract. Now we make concrete choices: which register holds which value, where things go in memory, how to pass arguments, how to call functions. These decisions are constrained by the architecture, not by our design preferences.

The Problem

Here's a piece of IR our compiler might produce:

main:
  entry:
    const %0, 42
    ret %0

We need to turn this into something like:

main:
  push  rbp
  mov   rbp, rsp
  mov   rax, 42
  mov   rsp, rbp
  pop   rbp
  ret

We need to solve a stack of sub-problems:

  • Virtual vs. physical registers: How do we map %0 to rax?
  • Memory management: Where do local variables live?
  • Control flow: How do basic blocks become labels and jumps?
  • Function calls: How do we pass arguments?

x86-64: A Quick Primer

We're targeting x86-64 in 64-bit mode. Here's what you need to know.

The Stack Frame

When a function runs, it sets up a stack frame. The convention we follow (System V AMD64 ABI): 

; Prologue
push rbp          ; save caller's base pointer
mov  rbp, rsp     ; our frame starts here
sub  rsp, <N>     ; allocate N bytes for locals

; ... function body ...

; Epilogue
mov  rsp, rbp     ; restore stack pointer
pop  rbp          ; restore caller's base pointer
ret               ; return to caller

After the prologue, every local variable lives at a negative offset from rbp: [rbp-8], [rbp-16], and so on. The frame size N must be a multiple of 16 to maintain stack alignment.

Strategy: Spill Everything

The simplest possible register allocation strategy is: don't use registers at all. Put every virtual register on the stack. Access them through rbp-relative loads and stores.

This is called "spilling everything." It's naive — every operation becomes a load-compute-store sequence — but it's correct, and it's the foundation every real register allocator improves upon.

StackAllocator

We walk the IR, find every virtual register, and assign each a unique stack slot. In OCaml, we use a Map.Make(Int) functor to get a type-safe map from virtual registers to offsets:

codegen.ml — StackAllocator
module VRegMap = Map.Make(Int)

type stack_alloc = {
  offsets: int VRegMap.t;
  frame_size: int;
}

let create () = { offsets = VRegMap.empty; frame_size = 0 }

let allocate alloc vreg =
  match VRegMap.find_opt vreg alloc.offsets with
  | Some offset -> (alloc, offset)
  | None ->
    let frame_size = alloc.frame_size + 8 in
    let offset = -frame_size in
    let offsets = VRegMap.add vreg offset alloc.offsets in
    ({ offsets; frame_size }, offset)

let frame_size alloc = alloc.frame_size
src/codegen.rs — StackAllocator
use std::collections::HashMap;

pub struct StackAllocator {
  offsets: HashMap<u32, i32>,
  frame_size: i32,
}

impl StackAllocator {
  pub fn new() -> Self {
    Self { offsets: HashMap::new(), frame_size: 0 }
  }

  pub fn allocate(&mut self, vreg: u32) -> i32 {
    if let Some(&offset) = self.offsets.get(&vreg) {
      return offset;
    }
    self.frame_size += 8;
    let offset = -self.frame_size;
    self.offsets.insert(vreg, offset);
    offset
  }

  pub fn frame_size(&self) -> i32 {
    self.frame_size
  }
}

Linear Scan Register Allocation

"Spill everything" produces terrible code. A better approach is linear scan register allocation, which assigns virtual registers to physical registers when possible and only spills when necessary.

The algorithm is surprisingly simple:

  1. Compute the live interval for each virtual register (first use to last use).
  2. Walk intervals in order of start point.
  3. Maintain a set of free physical registers.
  4. When an interval starts, assign a free register. If none is free, spill the interval that ends latest.
  5. When an interval ends, free its register.

In OCaml, this maps naturally to operations on lists and sets. The Set.Make(Int) functor gives us type-safe register sets:

codegen.ml — linear scan (simplified)
module RegSet = Set.Make(Int)

type interval = {
  vreg: int;
  start_: int;
  end_: int;
}

let linear_scan intervals num_phys_regs =
  let sorted = List.sort (fun a b -> Int.compare a.start_ b.start_) intervals in
  let free = ref (RegSet.of_list (List.init num_phys_regs Fun.id)) in
  let active = ref [] in
  let assignment = ref VRegMap.empty in

  List.iter (fun interval ->
    (* Expire old intervals *)
    active := List.filter (fun (_, end_) -> end_ > interval.start_)
      (!active);
    List.iter (fun (reg, _) -> free := RegSet.add reg (!free))
      (List.filter (fun (_, end_) -> end_ <= interval.start_)
        (!active));

    (* Assign or spill *)
    if not (RegSet.is_empty (!free)) then begin
      let reg = RegSet.min_elt (!free) in
      free := RegSet.remove reg (!free);
      active := (reg, interval.end_) :: !active;
      assignment := VRegMap.add interval.vreg reg (!assignment)
    end else begin
      (* Spill: no free register *)
      spill interval
    end
  ) sorted;
  !assignment
src/codegen.rs — linear scan (simplified)
use std::collections::{HashMap, HashSet};

struct Interval { vreg: u32, start: usize, end: usize }

fn linear_scan(
  intervals: Vec<Interval>,
  num_phys: usize,
) -> HashMap<u32, usize> {
  let mut sorted = intervals;
  sorted.sort_by_key(|i| i.start);

  let mut free: HashSet<usize> = (0..num_phys).collect();
  let mut active: Vec<(usize, usize)> = vec![];
  let mut assignment = HashMap::new();

  for interval in &sorted {
    // Expire old intervals
    active.retain(|&(reg, end)| {
      if end <= interval.start {
        free.insert(reg);
        false
      } else { true }
    });

    if let Some(&reg) = free.iter().next() {
      free.remove(&reg);
      active.push((reg, interval.end));
      assignment.insert(interval.vreg, reg);
    } else {
      spill(interval); // spill to stack
    }
  }
  assignment
}

OCaml's Pattern Matching Makes Instruction Selection Read Like a Specification

The heart of code generation is instruction selection — for each IR instruction, emit the corresponding x86-64 instructions. In OCaml, this is a single match expression where each case reads like a line in a specification document:

codegen.ml — instruction selection
type asm_emitter = {
  mutable buf: string list;
}

let emit e s = e.buf <- s :: e.buf
let emit_line e s = e.buf <- (s ^ "\n") :: e.buf

let slot_offset slot = Printf.sprintf "[rbp-%d]" ((slot + 1) * 8)
let reg_name r = Printf.sprintf "r%d" r  (* we map vregs to r0..rN *)

let emit_instr e alloc = function
  | Const (d, v) ->
    let offset = allocate alloc d in
    emit_line e (Printf.sprintf "  mov rax, %Ld" v);
    emit_line e (Printf.sprintf "  mov %s, rax" (slot_offset offset))

  | Add (d, a, b) ->
    let da = allocate alloc a in
    let db = allocate alloc b in
    let dd = allocate alloc d in
    emit_line e (Printf.sprintf "  mov rax, %s" (slot_offset da));
    emit_line e (Printf.sprintf "  add rax, %s" (slot_offset db));
    emit_line e (Printf.sprintf "  mov %s, rax" (slot_offset dd))

  | Sub (d, a, b) ->
    let da = allocate alloc a in
    let db = allocate alloc b in
    let dd = allocate alloc d in
    emit_line e (Printf.sprintf "  mov rax, %s" (slot_offset da));
    emit_line e (Printf.sprintf "  sub rax, %s" (slot_offset db));
    emit_line e (Printf.sprintf "  mov %s, rax" (slot_offset dd))

  | Mul (d, a, b) ->
    let da = allocate alloc a in
    let db = allocate alloc b in
    let dd = allocate alloc d in
    emit_line e (Printf.sprintf "  mov rax, %s" (slot_offset da));
    emit_line e (Printf.sprintf "  imul rax, %s" (slot_offset db));
    emit_line e (Printf.sprintf "  mov %s, rax" (slot_offset dd))

  | Div (d, a, b) ->
    let da = allocate alloc a in
    let db = allocate alloc b in
    let dd = allocate alloc d in
    emit_line e (Printf.sprintf "  mov rax, %s" (slot_offset da));
    emit_line e ("  xor rdx, rdx");
    emit_line e (Printf.sprintf "  mov rcx, %s" (slot_offset db));
    emit_line e ("  div rcx");
    emit_line e (Printf.sprintf "  mov %s, rax" (slot_offset dd))

  | Eq (d, a, b) ->
    emit_cmp e alloc a b;
    let dd = allocate alloc d in
    emit_line e ("  sete al");
    emit_line e (Printf.sprintf "  movzx rax, al");
    emit_line e (Printf.sprintf "  mov %s, rax" (slot_offset dd))

  | Lt (d, a, b) ->
    emit_cmp e alloc a b;
    let dd = allocate alloc d in
    emit_line e ("  setl al");
    emit_line e (Printf.sprintf "  movzx rax, al");
    emit_line e (Printf.sprintf "  mov %s, rax" (slot_offset dd))

  | Load (d, slot) ->
    let dd = allocate alloc d in
    emit_line e (Printf.sprintf "  mov rax, %s" (slot_offset slot));
    emit_line e (Printf.sprintf "  mov %s, rax" (slot_offset dd))

  | Store (slot, v) ->
    let dv = allocate alloc v in
    emit_line e (Printf.sprintf "  mov rax, %s" (slot_offset dv));
    emit_line e (Printf.sprintf "  mov %s, rax" (slot_offset slot))

  | Call (d, name, args) ->
    (* Push args in reverse order *)
    List.iter (fun vreg ->
      let dv = allocate alloc vreg in
      emit_line e (Printf.sprintf "  mov rax, %s" (slot_offset dv));
      emit_line e ("  push rax")
    ) (List.rev args);
    emit_line e (Printf.sprintf "  call %s" name);
    emit_line e (Printf.sprintf "  add rsp, %d" (List.length args * 8));
    let dd = allocate alloc d in
    emit_line e (Printf.sprintf "  mov %s, rax" (slot_offset dd))

  | Not (d, a) ->
    let da = allocate alloc a in
    let dd = allocate alloc d in
    emit_line e (Printf.sprintf "  mov rax, %s" (slot_offset da));
    emit_line e ("  not rax");
    emit_line e (Printf.sprintf "  mov %s, rax" (slot_offset dd))

  | Neg (d, a) ->
    let da = allocate alloc a in
    let dd = allocate alloc d in
    emit_line e (Printf.sprintf "  mov rax, %s" (slot_offset da));
    emit_line e ("  neg rax");
    emit_line e (Printf.sprintf "  mov %s, rax" (slot_offset dd))

struct AsmEmitter {
  buf: String,
}

impl AsmEmitter {
  fn emit(&mut self, s: &str) { self.buf.push_str(s); }
  fn emit_line(&mut self, s: &str) {
    self.buf.push_str(s); self.buf.push('\n');
  }
}

fn slot_offset(slot: usize) -> String {
  format!("[rbp-{}]", (slot + 1) * 8)
}

fn emit_instr(
  e: &mut AsmEmitter,
  alloc: &mut StackAllocator,
  instr: &IrInstr,
) {
  match instr {
    IrInstr::Const(d, v) => {
      let off = alloc.allocate(*d);
      e.emit_line(&format!("  mov rax, {}", v));
      e.emit_line(&format!("  mov {}, rax", slot_offset(off)));
    }
    IrInstr::Add(d, a, b) => {
      let da = alloc.allocate(*a);
      let db = alloc.allocate(*b);
      let dd = alloc.allocate(*d);
      e.emit_line(&format!("  mov rax, {}", slot_offset(da)));
      e.emit_line(&format!("  add rax, {}", slot_offset(db)));
      e.emit_line(&format!("  mov {}, rax", slot_offset(dd)));
    }
    IrInstr::Sub(d, a, b) => {
      let da = alloc.allocate(*a);
      let db = alloc.allocate(*b);
      let dd = alloc.allocate(*d);
      e.emit_line(&format!("  mov rax, {}", slot_offset(da)));
      e.emit_line(&format!("  sub rax, {}", slot_offset(db)));
      e.emit_line(&format!("  mov {}, rax", slot_offset(dd)));
    }
    IrInstr::Eq(d, a, b) => {
      emit_cmp(e, alloc, *a, *b);
      let dd = alloc.allocate(*d);
      e.emit_line("  sete al");
      e.emit_line("  movzx rax, al");
      e.emit_line(&format!("  mov {}, rax", slot_offset(dd)));
    }
    // ... remaining cases follow the same pattern
  }
}

Emitting a Complete Function

Now we put it all together. For each IR function, we emit the prologue, translate every block's instructions, handle the terminator, and emit the epilogue:

codegen.ml — function emission
let emit_func e alloc func =
  (* Function label *)
  emit_line e (Printf.sprintf "global %s" func.name);
  emit_line e (Printf.sprintf "%s:" func.name);

  (* Prologue *)
  emit_line e ("  push rbp");
  emit_line e ("  mov rbp, rsp");

  (* Emit each block *)
  List.iter (fun block ->
    emit_line e (Printf.sprintf ".%s:" block.label);
    List.iter (emit_instr e alloc) block.instrs;

    (* Terminator *)
    (match block.term with
    | Ret v ->
      let dv = allocate alloc v in
      emit_line e (Printf.sprintf "  mov rax, %s" (slot_offset dv));
      emit_line e ("  mov rsp, rbp");
      emit_line e ("  pop rbp");
      emit_line e ("  ret")
    | Br l ->
      emit_line e (Printf.sprintf "  jmp .%s" l)
    | BrCond (c, t, f) ->
      let dc = allocate alloc c in
      emit_line e (Printf.sprintf "  mov rax, %s" (slot_offset dc));
      emit_line e ("  cmp rax, 0");
      emit_line e (Printf.sprintf "  jne .%s" t);
      emit_line e (Printf.sprintf "  jmp .%s" f))
  ) func.blocks;

  (* Frame size allocation *)
  let frame = frame_size alloc in
  (* Align to 16 bytes *)
  let aligned = ((frame + 15) / 16) * 16 in
  (* Insert frame allocation after push rbp *)
  (* (In a real emitter, we'd track this up front and emit it inline) *)
  (fun () -> ())
src/codegen.rs — function emission
fn emit_function(
  e: &mut AsmEmitter,
  alloc: &mut StackAllocator,
  func: &IrFunction,
) {
  e.emit_line(&format!("global {}", func.name));
  e.emit_line(&format!("{}:", func.name));

  // Prologue
  e.emit_line("  push rbp");
  e.emit_line("  mov rbp, rsp");

  // Emit each block
  for block in &func.blocks {
    e.emit_line(&format!(".{}:", block.label));
    for instr in &block.instrs {
      emit_instr(e, alloc, instr);
    }
    match &block.term {
      IrTerm::Ret(v) => {
        let dv = alloc.allocate(*v);
        e.emit_line(&format!("  mov rax, {}", slot_offset(dv)));
        e.emit_line("  mov rsp, rbp");
        e.emit_line("  pop rbp");
        e.emit_line("  ret");
      }
      IrTerm::Br(l) => {
        e.emit_line(&format!("  jmp .{}", l));
      }
      IrTerm::BrCond(c, t, f) => {
        let dc = alloc.allocate(*c);
        e.emit_line(&format!("  mov rax, {}", slot_offset(dc)));
        e.emit_line("  cmp rax, 0");
        e.emit_line(&format!("  jne .{}", t));
        e.emit_line(&format!("  jmp .{}", f));
      }
    }
  }
}

Complete Example: func main(i32) { return 42; }

Let's trace the full codegen pipeline for our simplest program. The IR is: 

main:
  entry:
    %0 = const 42
    ret %0

The code generator produces: 

global main
main:
  push rbp
  mov  rbp, rsp
  sub  rsp, 8       ; allocate one stack slot (%0)
  mov  rax, 42
  mov  [rbp-8], rax   ; store %0
  mov  rax, [rbp-8]   ; load %0 for return
  mov  rsp, rbp
  pop  rbp
  ret

The assembly is verbose — every value goes through the stack — but it's correct. And correctness is the foundation. We can optimize later.

  mov  rcx, [rbp-16] ; load b

This is terrible code — and that's okay! Every unnecessary load and store is an opportunity for optimization. The register allocator will eliminate many of them. The key is that the generated code is correct. We can make it fast incrementally.

Calling Convention Details

The System V AMD64 ABI calling convention governs how functions pass arguments and return values. When our code generator emits a call, it must follow these rules:

  • Integer arguments 1-6 go in rdi, rsi, rdx, rcx, r8, r9 (in that order).
  • Additional arguments (7+) are pushed onto the stack in right-to-left order.
  • Return value goes in rax.
  • Stack alignment: RSP must be 16-byte aligned before the call instruction.
  • Caller-saved registers: rax, rcx, rdx, rsi, rdi, r8-r11 — the caller must save them if needed after the call.
  • Callee-saved registers: rbx, rbp, r12-r15 — the callee must save and restore them if used.

Our current "spill everything" strategy handles this trivially: everything is on the stack, so no registers need saving. But when we implement proper register allocation, we'll need to track which registers are live across calls and spill them around call sites. This is where liveness analysis becomes essential.

In OCaml, we model calling convention knowledge as a module with a clean interface:

Complete Walkthrough: while Loop Codegen

Let's trace the full codegen for a function with a loop: 

func sum_to_n(i32, n: i32) {
  let i: i32 = 0;
  let total: i32 = 0;
  while (i < n) {
    total = total + i;
    i = i + 1;
  }
  return total;
}

The IR builder produces a control flow graph with 4 blocks: entry, header, body, and after. The code generator then emits: 

global sum_to_n
sum_to_n:
  push rbp
  mov  rbp, rsp
  sub  rsp, 32       ; 4 vregs × 8 bytes

  ; entry: i = 0, total = 0
  mov  rax, 0
  mov  [rbp-8], rax   ; store i
  mov  rax, 0
  mov  [rbp-16], rax  ; store total
  jmp  .header

  ; header: check i < n
  .header:
  mov  rax, [rbp-8]   ; load i
  mov  rcx, [rbp-24]  ; load n
  cmp  rax, rcx
  setl al
  movzx rax, al
  mov  [rbp-32], rax  ; store comparison
  cmp  rax, 0
  jne  .body
  jmp  .after

  ; body: total += i; i++
  .body:
  mov  rax, [rbp-16]  ; load total
  add  rax, [rbp-8]   ; add i
  mov  [rbp-16], rax  ; store total
  mov  rax, [rbp-8]   ; load i
  add  rax, 1
  mov  [rbp-8], rax   ; store i
  jmp  .header

  ; after: return total
  .after:
  mov  rax, [rbp-16]  ; load total
  mov  rsp, rbp
  pop  rbp
  ret

The pattern is clear: every variable access becomes a load-modify-store sequence through rax. A real register allocator would keep i and total in registers r12 and r13, eliminating all those memory operations. But the code is correct — and correctness is our foundation.

The "Aha" Moment

Here's what we've done: we've crossed the final abstraction boundary. The IR — with its virtual registers and basic blocks — has become real assembly code that a CPU can execute. Every IR instruction maps to one or more x86-64 instructions. Every virtual register becomes a stack slot or a physical register. Every basic block becomes a labeled section of assembly.

In OCaml, the instruction selection function is a testament to the power of pattern matching. The match expression reads like a spec: "For Add(d, a, b), emit mov, add, mov." There's no ceremony, no visitor pattern, no method dispatch. Just a direct, exhaustive mapping from IR to assembly.

Exercises

1. Improve the stack allocator

Our stack allocator assigns 8 bytes to every virtual register. But booleans only need 1 byte. Modify the allocator to track the size of each value and allocate only as many bytes as needed. This reduces frame size and memory traffic.

2. Full linear scan register allocation

Replace the "spill everything" strategy with full linear scan register allocation. Implement live interval computation, register assignment, and spill code insertion. Test it on functions with many virtual registers.

3. Peephole optimization on the assembly

Add a peephole pass that scans the emitted assembly for redundant patterns. For example, mov rax, [rbp-8]; mov [rbp-8], rax is a no-op. Remove it. Other patterns: consecutive mov instructions to the same register, or cmp rax, 0; jne (can be jne with rax test).

4. Argument passing via registers

The System V ABI passes the first 6 integer arguments in registers (rdi, rsi, rdx, rcx, r8, r9). Modify the code generator to use registers for function arguments instead of pushing everything onto the stack. This is a major performance win.

5. Add floating-point support

Add f64 type support to the compiler. This requires new IR instructions (FAdd, FSub, etc.), new codegen patterns using SSE registers (xmm0-xmm15), and new type checking rules. The OCaml match makes adding new instruction variants straightforward — just add a new constructor and a new case.
