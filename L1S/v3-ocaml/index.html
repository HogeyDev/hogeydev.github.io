Building a Compiler — OCaml Edition

A hands-on, first-principles journey through compiler construction, with all code shown in OCaml.

What is this? This is a textbook that builds a complete compiler from the ground up, one chapter at a time. We don't start with a grand design. We start with a simple question — "how does a computer understand my code?" — and follow the thread wherever it leads. Each chapter solves one concrete problem. The next chapter introduces the problem that the previous solution creates. By the end, we'll have a working compiler in OCaml and a deep understanding of how it works.

Chapters

  1. The SparkWhy compilers? What will we build? The big picture.
  2. Raw MaterialsSource code as text. Positions, spans, reading files.
  3. Finding the WordsBreaking text into tokens. Keywords, identifiers, literals.
  4. Making Sense of StructureFrom flat tokens to nested trees. Recursive descent parsing.
  5. The Shape of a ProgramThe Abstract Syntax Tree. What the parser really builds.
  6. Does This Make Sense?Type checking. Symbol tables. Catching mistakes.
  7. Bridging the GapAn intermediate language between AST and assembly.
  8. Speaking the Machine's LanguageGenerating x86-64 assembly. Registers, stack frames, calling conventions.
  9. The Final MileDriving the pipeline. Error reporting. Assembling and linking.
  10. Where Next?What we built. What comes next. Further exploration.
