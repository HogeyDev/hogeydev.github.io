Building a Compiler OCaml Edition

8 · The Final Mile

We can generate assembly text from source code. But assembly text isn't an executable. It's a string — characters in a file. How do we cross that final gap from .asm to a program that actually runs?

The answer involves two tools we don't build ourselves: an assembler (NASM) and a linker (cc/ld). Our job is to orchestrate them. We become the conductor of a pipeline that starts with a .spark source file and ends with a binary that the OS can load and execute.

The Driver Pipeline

The driver is the top-level orchestrator. It connects every piece of the compiler into a single coherent flow:

Source file (.spark) │ ▼ 1. Read source into string │ ▼ 2. Lex → token list │ ▼ 3. Parse → AST │ ▼ 4. Type check → annotated AST │ ▼ 5. Build IR → ir_module │ ▼ 6. Codegen → assembly string │ ▼ 7. Write assembly to .asm file │ ▼ 8. Invoke NASM: nasm -f elf64 file.asm -o file.o │ ▼ 9. Invoke linker: cc -o file file.o -no-pie │ ▼ Executable (file)

Error Handling Strategy

A good compiler doesn't stop at the first error. It collects as many errors as possible, then presents them to the user all at once. Our OCaml compiler uses a simple pattern: each phase collects errors in a mutable list, and the driver checks the error count before proceeding to the next phase.

Read source → string │ ▼ Lex → tokens (collect errors) │ succ │ fail ▼ ▼ Parse → AST Print errors & exit │ succ │ fail ▼ ▼ Type check → AST Print errors & exit │ succ │ fail ▼ ▼ Build IR → module Print errors & exit │ succ ▼ Codegen → assembly string │ ▼ Write .asm, assemble, link
Error-aware types in OCaml In OCaml, we use ('a, error list) Result.t for operations that can fail, or simply maintain a mutable errors: error list field in each phase object. Multi-error accumulation is crucial: finding all bugs in one compile pass is much faster than fixing one at a time.

The Driver in OCaml

Here's the main function of the compiler, connecting every piece. OCaml's Unix module provides process management, and Printf / eprintf handle output:

main.ml — complete driver
let read_file path =
  let ch = open_in path in
  let n = in_channel_length ch in
  let s = really_input_string ch n in
  close_in ch;
  s

let report_errors file errors =
  List.iter (fun e ->
    Errors.report file e.span e.msg
  ) errors;
  exit 1

let () =
  let args = Sys.argv in
  if Array.length args < 2 then begin
    eprintf "usage: %s <file.spark>\n" args.(0);
    exit 1
  end;

  let path = args.(1) in

  (* 1. Read source *)
  let source = try read_file path
    with e -> eprintf "error reading %s: %s\n" path
                  (Printexc.to_string e); exit 1
  in
  let file = Span.make_source_file path source in

  (* 2. Lex *)
  let lexer = Lexer.create file in
  let tokens = Lexer.lex lexer in
  if lexer.errors <> [] then
    report_errors file lexer.errors;

  (* 3. Parse *)
  let parser = Parser.create tokens file in
  let ast = Parser.parse parser in
  if parser.errors <> [] then
    report_errors file parser.errors;

  (* 4. Symbol table + Type check *)
  let symbols = Symbol.collect_decls ast in
  let checker = Check.create symbols in
  let typed_ast = Check.check_program checker ast in
  if checker.errors <> [] then
    report_errors file checker.errors;

  (* 5. Build IR *)
  let ir_builder = IrBuilder.create () in
  let ir_module = IrBuilder.build_module ir_builder typed_ast in
  if ir_builder.errors <> [] then
    report_errors file ir_builder.errors;

  (* 6. Codegen *)
  let emitter = Emitter.create () in
  Codegen.codegen_module emitter ir_module;

  (* 7. Write assembly *)
  let asm_path = Filename.chop_extension path ^ ".asm" in
  Emitter.write_to_file emitter asm_path;
  printf "wrote %s\n" asm_path;

  (* 8. Assemble with NASM *)
  let obj_path = Filename.chop_extension path ^ ".o" in
  let nasm_status = Sys.command
    (Printf.sprintf "nasm -f elf64 %s -o %s" asm_path obj_path)
  in
  if nasm_status <> 0 then exit 1;

  (* 9. Link with cc *)
  let bin_path = Filename.chop_extension path in
  let link_status = Sys.command
    (Printf.sprintf "cc -o %s %s -no-pie" bin_path obj_path)
  in
  if link_status <> 0 then exit 1;

  printf "built %s\n" bin_path
src/main.rs — complete driver
use std::env;
use std::fs;
use std::process;

mod span;
mod token;
mod lexer;
mod ast;
mod parser;
mod symbol;
mod types;
mod check;
mod ir;
mod codegen;
mod emitter;

fn main() {
  let args: Vec<String> = env::args().collect();
  if args.len() < 2 {
    eprintln!("usage: {} <file.spark>", args[0]);
    process::exit(1);
  }

  let path = &args[1];
  let source = fs::read_to_string(path)
    .unwrap_or_else(|e| { eprintln!("error: {}", e); process::exit(1); });

  let file = span::SourceFile::new(path, &source);
  let mut errors: Vec<span::Error> = vec![];

  let mut lexer = lexer::Lexer::new(&file);
  let tokens = lexer.lex();
  if !lexer.errors.is_empty() { report_and_exit(&file, &lexer.errors); }

  let mut parser = parser::Parser::new(&tokens, &file);
  let ast = parser.parse();
  if !parser.errors.is_empty() { report_and_exit(&file, &parser.errors); }

  let symbols = symbol::build(&ast);
  let mut checker = check::Checker::new(symbols);
  let ast = checker.check(ast);
  if !checker.errors.is_empty() { report_and_exit(&file, &checker.errors); }

  let mut ir_builder = ir::Builder::new();
  let ir_module = ir_builder.build(&ast);
  if !ir_builder.errors.is_empty() { report_and_exit(&file, &ir_builder.errors); }

  let mut emitter = emitter::Emitter::new();
  codegen::codegen_module(&ir_module, &mut emitter);

  let asm_path = path.replace(".spark", ".asm");
  emitter.write(&asm_path).unwrap();
  println!("wrote {}", asm_path);

  let obj_path = path.replace(".spark", ".o");
  let status = process::Command::new("nasm")
    .args(&["-f", "elf64", &asm_path, "-o", &obj_path])
    .status().expect("nasm not found");
  if !status.success() { process::exit(1); }

  let bin = path.replace(".spark", "");
  let status = process::Command::new("cc")
    .args(&["-o", &bin, &obj_path, "-no-pie"])
    .status().expect("cc not found");
  if !status.success() { process::exit(1); }

  println!("built {}", bin);
}

The Emitter Module

The emitter accumulates assembly lines and writes the result. In OCaml, this is a record with a mutable string list buffer:

emitter.ml
type t = {
  mutable lines: string list;
}

let create () = { lines = [] }

let emit e s = e.lines <- s :: e.lines

let emit_line e s = e.lines <- (s ^ "\n") :: e.lines

let to_string e =
  String.concat "" (List.rev e.lines)

let write_to_file e path =
  let oc = open_out path in
  output_string oc (to_string e);
  close_out oc
src/emitter.rs
use std::fs;

pub struct Emitter {
  buf: String,
}

impl Emitter {
  pub fn new() -> Self {
    Emitter { buf: String::new() }
  }

  pub fn emit(&mut self, s: &str) {
    self.buf.push_str(s);
  }

  pub fn emit_line(&mut self, s: &str) {
    self.buf.push_str(s);
    self.buf.push('\n');
  }

  pub fn write(&self, path: &str) -> Result<()> {
    fs::write(path, &self.buf)?;
    Ok(())
  }
}

Running the Compiler

With the driver complete, we can compile entire programs: 

$ cat hello.spark
func main(i32) {
  return 42;
}

$ dune exec -- ./spark hello.spark
wrote hello.asm
built hello

$ ./hello
$ echo $?
42

The exit code is 42. Our compiler works. From source code to running binary, through every phase we built together.

Testing the Compiler

A compiler needs tests at every level. Here's our testing strategy:

  • Unit tests for each phase: lex individual tokens, parse individual constructs, type-check individual expressions.
  • Integration tests that compile entire programs and check the exit code.
  • Snapshot tests that compare generated assembly against expected output.

In OCaml, we use dune runtest with inline tests or separate test files. Here's a simple test runner in OCaml:

test_runner.ml
let test_compile name source expected_exit =
  let file = Filename.temp_file name ".spark" in
  let oc = open_out file in
  output_string oc source;
  close_out oc;

  let asm = Filename.chop_extension file ^ ".asm" in
  let obj = Filename.chop_extension file ^ ".o" in
  let bin = Filename.chop_extension file in

  (* Run the compiler *)
  let status = Sys.command
    (Printf.sprintf "./spark %s >/dev/null 2>&1" file) in

  if status <> 0 then
    failwith (Printf.sprintf "compiler failed for %s" name);

  (* Run the binary *)
  let exit_code = Sys.command (Printf.sprintf "%s" bin) in
  if exit_code <> expected_exit then
    failwith (Printf.sprintf
      "%s: expected exit %d, got %d" name expected_exit exit_code);

  (* Cleanup *)
  Sys.remove file;
  Sys.remove asm;
  Sys.remove obj;
  Sys.remove bin

let () =
  test_compile "simple_return"
    "func main(i32) { return 42; }\n" 42;

  test_compile "add"
    "func main(i32) { return 2 + 3; }\n" 5;

  printf "All tests passed!\n"
tests/integration.rs
use std::process::Command;

fn test_compile(name: &str, source: &str, expected: i32) {
  let dir = format!("/tmp/{}", name);
  std::fs::create_dir_all(&dir).unwrap();
  let spark = format!("{}/test.spark", dir);
  std::fs::write(&spark, source).unwrap();

  let status = Command::new("cargo")
    .args(&["run", "--", &spark])
    .status().unwrap();
  assert!(status.success(), "{}: compiler failed", name);

  let bin = format!("{}/test", dir);
  let out = Command::new(&bin)
    .output().unwrap();
  assert!(out.status.code() == Some(expected),
    "{}: expected {}, got {:?}", name, expected, out.status.code());
}

fn main() {
  test_compile("simple_return", "func main(i32) { return 42; }\n", 42);
  test_compile("addition", "func main(i32) { return 2 + 3; }\n", 5);
  println!("All tests passed!");
}

The "Aha" Moment

Here's what we've accomplished: a complete, working compiler from source code to executable. Every phase is connected, every error is caught, every transformation is correct. The pipeline that was a diagram in Chapter 0 is now a running program on your computer.

In OCaml, the driver is refreshingly straightforward. Each phase is a module with a clean interface. The Sys.command call for NASM and cc is pragmatic — we don't need to reimplement assemblers and linkers. The combination of OCaml's module system and the Unix process model gives us a clean, composable pipeline.

From here, the compiler is an infinitely extensible platform. Add new types. Add new optimizations. Add a new backend. The foundation is solid.

Exercises

1. Add a --emit-ir flag

Add a command-line flag that dumps the IR instead of generating assembly. This is invaluable for debugging: ./spark --emit-ir test.spark prints the IR without running NASM or cc.

2. Multiple source files

Extend the driver to accept multiple .spark files and compile them together. Each file is lexed and parsed independently, but they share a single symbol table and IR module. What changes to the pipeline does this require?

3. Error message improvements

Add colored output to error messages. Use ANSI escape codes to highlight the error location in red. Make the ~ underline point exactly at the problematic token.

4. Inline assembly

Add an asm("...") expression that passes its string directly to the assembly output. This is useful for low-level operations that the compiler doesn't support natively. How does this interact with the register allocator?

5. Caching compilation results

Implement a simple cache that skips recompilation if the source file hasn't changed. Store a hash of the source alongside the compiled binary. Skip the pipeline if the hash matches.
